# Configuration

- `default.yaml`: shared developer defaults
- `local.yaml`: machine-specific developer overrides, generated by `bootstrap.sh` and ignored by Git
- `model-catalog.yaml`: shared model metadata, profiles, licenses
- `model-catalog.local.yaml`: installed paths and validation state, generated locally and ignored by Git
- `episodes/`: one episode per YAML file
- `effects.yaml`: separately selected presentation effects and rarity upgrades

Application settings saved by operators live at
`data/runtime/staff-settings.json` and apply from the next session.
